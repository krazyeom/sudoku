# ZenGrid Sudoku

모던 스타일의 수도쿠 웹앱입니다. 퍼즐 생성, 자동입력/힌트, 기록 저장, 공유 카드, 그리고 **실시간 멀티플레이(방 기반 동기화)** 를 지원합니다.

## 주요 기능

- **난이도 3단계**: 하 / 중 / 상
- **메모 모드**와 **자동입력 / 힌트**
- **기록 저장/필터/정렬/내보내기/가져오기**
- **완료 시 공유 카드** 생성
- **실시간 멀티플레이**: 같은 방에 접속한 사용자끼리 보드가 동기화됨
- **PM2 운영 스크립트** 제공

## 요구 사항

- Node.js 18 이상 권장
- npm

## 설치

```bash
npm install
```

## 실행

### 개발/로컬 실행

```bash
npm run dev
```

기본 주소:

- `http://localhost:6767`

### 프로덕션 실행

```bash
npm run build
npm start
```

### PM2로 실행

```bash
bash scripts/start.sh
```

중지:

```bash
bash scripts/stop.sh
```

## 조작 방법

- 숫자 **1~9**: 선택한 칸에 숫자 입력
- **Backspace / Delete / 0**: 칸 지우기
- **Esc**: 선택 해제
- **N**: 메모 모드 토글
- **I**: 힌트 소모 후 후보 칸 표시
- **H**: 자동입력 아이템 사용
- **검사** 버튼: 현재 보드가 해답과 맞는지 검사
- **Undo** 버튼: 최근 입력 되돌리기

## 멀티플레이 사용법

이 앱의 멀티플레이는 **방(room)** 단위로 동작합니다. 방에 연결된 모든 사용자는 같은 퍼즐 상태를 공유합니다.

### 1) 방 만들기

1. 앱을 연다.
2. 왼쪽 패널의 **방 만들기** 버튼을 누른다.
3. 서버가 새 방을 생성하고, 현재 주소에 `?room=방ID` 를 붙여줍니다.
4. **방 ID** 버튼을 누르면 방 번호를 바로 복사할 수 있다.
5. **초대 링크 복사** 버튼으로 공유 링크를 복사한다.
6. 그 링크를 다른 사람에게 보내면 된다.

### 2) 방 참가하기

참가하는 방법은 2가지입니다.

#### 방법 A. 초대 링크로 참가

- 방장이 보낸 링크를 그대로 열면 자동으로 방에 참가합니다.
- 예시:

```text
http://서버주소:6767/?room=ROOM_ID
```

#### 방법 B. 방 ID를 직접 입력해서 참가

1. 앱을 연다.
2. 왼쪽 패널의 **방 ID** 입력칸에 값을 넣는다.
3. **방 참가** 버튼을 누른다.
4. 방이 존재하면 바로 동기화된다.
5. 붙여넣은 값이 전체 링크여도 `room=` 값만 자동으로 추출된다.

### 3) 멀티플레이에서 동기화되는 것

- 숫자 입력 / 삭제
- 메모 상태
- 퍼즐 완료 상태
- 방 재시작(방장만 가능)

### 4) 역할

- **방장(host)**: 새 퍼즐 시작 가능
- **참가자(guest)**: 퍼즐을 함께 풂
- **관전자(spectator)**: 상태 확인 중심

### 5) 종료 / 나가기

- **방 나가기** 버튼을 누르면 현재 방에서 빠져나옵니다.
- 브라우저 탭을 닫아도 연결은 해제됩니다.

## 환경 변수

### Production / Home Linux server

기본적으로 같은 호스트의 `/ws` 로 연결합니다. 이 프로젝트는 **HTTP 전용**으로 운영해도 됩니다.

- `NEXT_PUBLIC_WEBSOCKET_URL`
  - WebSocket 주소, 예: `ws://192.168.0.10:6767/ws` 또는 `ws://example.com/ws`
  - `http://...` 를 넣어도 내부에서 `ws://...` 로 변환됩니다.

### Local Development

`npm start` 또는 `bash scripts/start.sh` 로 띄우면 웹소켓 서버가 함께 올라갑니다.
`npm run dev` 도 이제 같은 custom server를 사용합니다.

### 주의 사항

- 공유 방 보드는 두 플레이어가 모두 연결될 때까지 숨겨집니다.
- 두 플레이어가 준비되면 5초 카운트다운 후 같은 보드가 동시에 공개됩니다.

## 운영 팁

- 멀티플레이를 쓰려면 **같은 서버 주소**로 접속해야 합니다.
- 로컬에서는 같은 와이파이/내부망에서 같은 IP와 포트를 사용하면 됩니다.
- 서버가 HTTPS 뒤에 있다면 WebSocket도 자동으로 `wss://` 로 연결됩니다.

## 문제 해결

### CSS가 깨져 보일 때

1. `npm run build` 를 다시 실행한다.
2. `npm start` 또는 `bash scripts/start.sh` 로 프로덕션 서버를 다시 띄운다.
3. 브라우저 캐시를 강력 새로고침한다.

### 방에 연결되지 않을 때

- URL의 `room` 값이 맞는지 확인한다.
- 방장이 서버를 재시작했다면 새 초대 링크를 받아야 한다.
- 서로 다른 서버/포트에 접속하면 같은 방이라도 연결되지 않는다.
